About

What is Open Design?

A complete design platform that runs locally alongside your coding agent.

31 Skills, 72 Design Systems

Ship with a rich skill library covering prototypes, decks, mobile apps, dashboards, email marketing, social carousels, and more. Each backed by one of 72 curated brand-grade design systems.

Sandboxed Preview + Export

Every artifact renders in a clean srcdoc iframe. Editable in place via the file workspace; downloadable as HTML, PDF, PPTX, ZIP, or Markdown. No external services needed.

10+ Coding Agent CLIs

Plug into Claude Code, Codex, Cursor, Gemini CLI, OpenCode, Qwen, Copilot, Hermes, Kimi CLI, Pi, or any OpenAI-compatible BYOK provider.

See It in Action

The complete design workflow, from brief to pixel-perfect output.

Entry View

Pick a skill, pick a design system, type the brief. The same surface for prototypes, decks, mobile apps, dashboards, and editorial pages.

Turn-1 Discovery Form

Before the model writes a pixel, OD locks the brief: surface, audience, tone, brand context, scale. 30 seconds of radios beats 30 minutes of redirects.

Direction Picker

When the user has no brand, the agent emits 5 curated directions (Monocle / Modern Minimal / Tech Utility / Brutalist / Soft Warm). One click = a deterministic palette + font stack.

Sandboxed Preview

Every artifact renders in a clean srcdoc iframe. Editable in place via the file workspace; downloadable as HTML, PDF, ZIP.

72-System Library

Every product system shows its 4-color signature. Click for the full DESIGN.md, swatch grid, and live showcase.

Deck Mode (guizang-ppt)

Magazine layouts, WebGL hero backgrounds, single-file HTML output, PDF export. The bundled guizang-ppt-skill drops in unchanged.

Mobile Prototype

Pixel-accurate iPhone 15 Pro chrome (Dynamic Island, status bar SVGs, home indicator). Multi-screen prototypes use shared /frames/ assets.

Architecture

How It Works

Three-tier local architecture: Web UI + Daemon + Coding Agent CLI.

+------------------------------------------------------------------+
|                         Browser (Next.js)                        |
|  chat . file workspace . iframe preview . settings . imports     |
+-------------------+----------------------------------------------+
                    | http://localhost:3000
+-------------------v----------------------------------------------+
|                     Daemon (Node.js)                             |
|  /api/chat/stream  /api/projects  /api/skills  /api/templates    |
|  /api/export       /api/design-systems  /api/proxy/stream        |
|  /api/artifacts/lint  /api/import/claude-design                  |
+-------------------+----------------------------------------------+
                    | spawn + stdio / OpenAI-compat SSE
+-------------------v----------------------------------------------+
|              Coding-Agent CLI  (user's choice)                   |
|  Claude Code | Codex | Gemini | OpenCode | Cursor | Qwen | ...  |
+------------------------------------------------------------------+
Web

Next.js App Router

Chat, composer with skill/design-system picker, question forms, direction picker, sandboxed iframe preview, file workspace, settings, Claude Design ZIP import.

Daemon

Node.js Background Process

Prompt-stack assembly (system + skill + design-system + user turns), agent lifecycle, project persistence, multi-format export (HTML/PDF/PPTX/ZIP/MD), BYOK proxy with SSRF blocklist.

Agent

Your Existing CLI

The daemon spawns whichever coding-agent CLI you have installed and bridges stdio or OpenAI-compatible SSE. No vendor lock-in.

Desktop

Optional Electron Shell

Sidecar IPC protocol (STATUS / EVAL / SCREENSHOT / CONSOLE / CLICK / SHUTDOWN) for headless E2E testing against a real Electron window.

Guide

Quick Start

Get Open Design running locally in under 2 minutes.

1

Clone and install

bash 
git clone https://github.com/nexu-io/open-design.git
cd open-design
npm install
2

Start the daemon + web app

bash 
npm run dev

Opens http://localhost:3000. The daemon auto-starts in the background.

3

Pick a skill + design system

The entry view lets you select from 31 skills and 72 design systems. Type your brief and hit send — the agent handles the rest.

4

Preview, edit, export

Artifacts render in the sandboxed preview. Edit inline, then export as HTML, PDF, PPTX, ZIP, or Markdown.

Contributing

Issues, PRs, new skills, and new design systems are all welcome. The highest-leverage contributions:

  • Add a skill — drop a folder into skills/ following the SKILL.md convention
  • Add a design system — drop a DESIGN.md into design-systems/<brand>/ using the 9-section schema
  • Wire up a new agent CLI — one entry in apps/daemon/src/agents.ts

Full walkthrough: CONTRIBUTING.md
